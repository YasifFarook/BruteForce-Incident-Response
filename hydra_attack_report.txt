 	SSH Brute-Force Attack Simulation using Hydra

Analysis of SSH Brute-Force Attack Using Hydra:

Overview
This project aimed to simulate an SSH brute-force attack using Hydra and a limited subset of the RockYou password list. The attack successfully identified the correct password, demonstrating the vulnerability of weak SSH credentials.

Findings
The attack was executed against 127.0.0.1 (localhost) with the username kali.
A subset of 500 passwords was used to limit execution time.
After 499 attempts, the correct password was identified.

Initial issues, such as connection errors and SSH rate limiting, were resolved by adjusting Hydra’s task settings and ensuring SSH was running properly.

Observations
- Connection Issues: The attack was initially slowed down by SSH protections, leading to errors like “Connection reset by peer.”
- Brute-Force Speed: Using -t 4 limited parallel attempts, avoiding detection mechanisms while maintaining efficiency.
- System Logging: No auth.log file was found, likely due to default logging configurations in Kali Linux.

Implications
- Real-world risks: Weak passwords make SSH servers vulnerable to brute-force attacks.
- Mitigation: Using strong, complex passwords, limiting SSH login attempts, and enabling fail2ban can prevent such attacks.
- Alternative Defenses: Disabling password authentication and using SSH keys would make brute-force attacks ineffective.

Attack Execution Details:

To simulate the brute-force attack:
hydra -t 1 -W 5 -l kali -P /tmp/rockyou_limited.txt ssh://127.0.0.1 -V > hydra_attack.log

To monitor progress of the attack:
tail -f hydra_attack.log

Results were logged onto hydra_attack.log. This data was then saved to hydra_attack_report.txt for investigative anaylsis.
